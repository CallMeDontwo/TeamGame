## 一、英雄技能 — 冰墙 (Ice Wall)

### 玩家体验目标
> 玩家在关键时刻召唤一堵冰墙，阻挡敌人的推进路线，争取时间调整阵型或集火落单的敌人。

### 效果描述
在指定位置生成一堵横向的冰墙，持续 `[DURATION]` 秒。冰墙不可移动，拥有独立的 `[HP]`。
- 敌人无法穿过冰墙（碰撞阻挡）
- 远程敌人的弹道可以被冰墙挡住
- 冰墙被摧毁后碎裂消失，对周围敌人造成减速效果

### 输入方式
- 点击英雄 → 按技能键 → 在战场上选择一个放置点
- 或者：在英雄前方固定距离自动生成，降低操作门槛

### 关键参数

| 参数 | 假设值 | 说明 |
|---|---|---|
| 持续时间 | 5 秒 `[PLACEHOLDER]` | 够阻挡一波小怪冲锋 |
| 冰墙 HP | `[PLACEHOLDER]` | 随技能等级提升 |
| 冰墙宽度 | 2 个身位 `[PLACEHOLDER]` | 横版战场的 1/4 左右 |
| 减速效果 | 移动速度 -50%，持续 2 秒 | 冰墙碎裂后触发 |
| 冷却时间 | 12 秒 `[PLACEHOLDER]` | 一场战斗能用 2-3 次 |

### 与现有系统的关系

```
英雄按下技能键
  ↓
选择放置位置（或自动放前方）
  ↓
AIBattleComponent.SpawnIceWall(x, y)
  ↓
生成一个"冰墙"实体 (Unit)
  ├── StateComponent (Idle → 不可移动)
  ├── 碰撞体 (阻挡敌人寻路/移动)
  ├── HP (可以被怪物攻击)
  └── 持续计时器 → 到期消失 → 触发碎裂减速
```

### 冰墙作为"单位"处理
冰墙本质上是一个特殊的 **Unit**：
- `UnitType = Special`（新增枚举值）或复用 `UnitType.NPC`
- `IdentityComponent.Name = "冰墙"`
- `StateComponent` 固定为 Idle，永远不切换
- 没有 `AIBattleComponent`（不需要 AI）
- 没有 `MoveComponent`（不能移动）
- 有碰撞体组件（需要新增 `ColliderComponent` 或直接用 Spine 碰撞）
- 有 HP，可被攻击

### 开放问题
- 冰墙是否占用英雄站位？如果怪物绕路，冰墙的战术价值是否打折扣？
- 是否需要区分"阻挡一切"和"只阻挡敌人"？
- 碎裂减速的范围多大？

## 二、怪物技能 — 锁定冲锋+爆炸 (Charge & Blast)

### 玩家体验目标
> 玩家看到一只怪物锁定城墙开始蓄力冲锋——必须在它到达之前集火干掉它，否则城墙会受到巨额伤害。这是一个"必须处理的威胁"，制造紧张感。

### 效果描述
怪物锁定城墙为目标，进行一段时间的蓄力（身上出现特效），然后沿直线向城墙冲锋。到达城墙后**自爆**，对城墙造成大量伤害，并对周围英雄造成范围伤害和击退。

### 触发条件
- 怪物生成时就带此技能
- 或者：怪物在进入战场 `[TIME]` 秒后自动激活
- 或者：当怪物 HP 低于 `[THRESHOLD]%` 时触发的狂暴行为

### 行为流程

```
怪物生成 / 进入战场
  ↓
锁定城墙为目标
  ↓
蓄力阶段 (2 秒)
  ├── 播放蓄力动画 (charge)
  ├── 身上闪烁/粒子特效
  └── 玩家可以在此期间击杀它来阻止冲锋
  ↓
冲锋阶段
  ├── 沿直线高速冲向城墙 (速度 * 3~5 倍)
  ├── 不可阻挡（霸体，免疫击退/眩晕）
  ├── 路径上碰到英雄 → 击飞并造成伤害
  └── 无法被重新选为目标（已经锁死了）
  ↓
到达城墙 → 爆炸
  ├── 对城墙造成 [DAMAGE] 伤害
  ├── 对周围 [RADIUS] 内的英雄造成范围伤害
  └── 怪物自身死亡（自爆）
```

### 关键参数

| 参数 | 假设值 | 说明 |
|---|---|---|
| 蓄力时间 | 2 秒 `[PLACEHOLDER]` | 给玩家反应时间 |
| 冲锋速度 | 3× 基础移速 `[PLACEHOLDER]` | 必须够快，形成压迫感 |
| 霸体 | 免疫控制 | 冲锋过程中不可阻挡 |
| 对城墙伤害 | `[PLACEHOLDER]` | 一次冲锋 ≈ 5 个小怪同时A城墙的伤害 |
| 爆炸范围 | 2 个身位 `[PLACEHOLDER]` | 波及英雄的范围 |
| 对英雄伤害 | `[PLACEHOLDER]` | 中高伤害，应被规避 |
| 冷却/触发频率 | 每 15 秒触发一次 `[PLACEHOLDER]` | 不会连续出现 |

### 与现有系统的关系

这个技能需要为怪物新增 **两个状态**（或扩展 `UnitState`）：

```
UnitState 新增枚举:
  Charging  = 7   // 蓄力中
  Dashing   = 8   // 冲锋中
```

怪物行为流程（AI 扩展）：

```
AIBattleComponent.Update()
  ├── 正常模式: Idle → Move → Attack (自走棋默认行为)
  └── 触发冲锋:
        ↓ 切换至特殊行为
       ChangeState(Charging)
        ├── 播放蓄力动画，持续 [CHARGE_TIME]
        ├── 锁定城墙坐标
        └── 计时结束 → ChangeState(Dashing)
              ├── 高速移向城墙 (硬编码移动，不使用 MoveComponent)
              ├── 到达城墙 → 爆炸
              │    ├── 城墙扣血
              │    ├── 范围伤害英雄
              │    └── 怪物自杀 (Dispose)
              └── 中途被击杀 → 直接死亡，不爆炸
```

### 视觉反馈要点
> 玩家需要一眼看出"这个怪物要冲锋了"
- 蓄力阶段：怪物身上闪烁红光、缩放抖动、地面出现红色目标指示线指向城墙
- 冲锋阶段：拖尾特效、地面烟尘
- 爆炸：屏幕震动、粒子爆发、城墙碎裂效果

### 开放问题
- 多个冲锋怪物同时出现时怎么处理？同时冲锋对城墙伤害是否叠加？
- 英雄是否有技能可以打断冲锋（如眩晕/击飞）？这决定了冲锋怪物的威胁层次。
- 冲锋怪物是否应该有易伤（蓄力阶段受到更多伤害）来奖励集火？
